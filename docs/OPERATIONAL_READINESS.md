# Operational Readiness Checklist

This checklist captures the minimum operational items required before accepting paying clients.

Priority sections (critical -> important):

1. Client Delivery System (Critical)
  - Standardized Packages defined and published (Starter, Business, Premium, Maintenance Only, Hosting+Maintenance)
  - Onboarding workflow implemented and tested (intake form, branding upload, domain collection, approvals, deployment)
  - Proposal generator ready and templated

2. Template System (High)
  - 3–5 reusable templates created and test-provisionable
  - Each template includes responsive layout, contact forms, SEO basics, CMS-ready hooks

3. Contracts & Legal (Urgent)
  - Service agreement, hosting agreement, maintenance agreement, cancellation and ownership clauses

4. Payment Flow (Very Important)
  - Invoice templates, downpayment rules (50% min), payment confirmation workflow, renewal tracking

5. Client Communication (Critical)
  - Support hours, SLA, official channels, emergency process, client portal for requests

6. Domain & Hosting Ops (Very Important)
  - Registrar access tracking, renewal schedule, SSL/backup policies, password manager usage

7. Sales System (Most Important)
  - 2–3 lead sources, outreach scripts, CRM stages mapped in the ops suite

8. Financial Tracking (Important)
  - MRR tracking, hosting costs, domain costs, profit per client

9. Backup Human Operations (Important)
  - Documented runbooks for deployments, env vars, backups, registrars, client credentials

10. Production Deployment Standards (Required)
  - Separate production Supabase and Vercel projects, SSL monitoring, backup validation, alerting

Quick Go/No-Go Criteria
- All critical items (1, 4, 5, 10) documented and one end-to-end staging run completed.
- Contracts exist and can be provided at signup.
- Payment flow validated for at least one channel.
