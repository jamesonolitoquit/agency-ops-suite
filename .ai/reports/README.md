# Reports

Generated outputs from scanners, workflows, and analysis commands.

## Purpose

Reports provide:
- **Visibility** — See project health metrics
- **Accountability** — Track progress over time
- **Actionability** — Prioritized recommendations
- **Trending** — Identify patterns and degradation
- **Documentation** — Historical record of decisions

Reports are generated artifacts, not manually created.

## Report Types

### Scanner Reports
- `security-report.md` — Security vulnerabilities and remediation
- `architecture-report.md` — Architecture violations and guidance
- `performance-report.md` — Performance audit findings
- `workspace-hygiene-report.md` — Code quality issues
- `accessibility-report.md` — WCAG compliance gaps
- `deployment-readiness-report.md` — Deployment blockers

### Analysis Reports
- `pr-review.md` — Pull request code review
- `feature-analysis.md` — Feature impact assessment
- `tech-debt-analysis.md` — Technical debt inventory
- `dependency-analysis.md` — Dependency health report

### Dashboard Reports
- `project-health.md` — Overall project metrics dashboard
- `weekly-summary.md` — Weekly progress and blockers
- `monthly-review.md` — Monthly retrospective and metrics

## Report Usage

**Daily Development:**
- Review `workspace-hygiene-report.md` before commit
- Check `security-report.md` after major changes

**Before Release:**
- Review all active reports
- Ensure critical issues resolved
- Verify deployment readiness

**Regular Reviews:**
- Weekly: Health dashboard
- Monthly: Trend analysis and retrospective

## Archiving Reports

- Generated reports stored with timestamp
- Keep recent reports (last 30 days) active
- Archive older reports for historical reference
- Never delete reports

## Report Generation

Reports are generated by:
- Automated scanners (scheduled or on-demand)
- CI/CD pipelines (on PR, pre-deploy)
- Manual command invocation
- Dashboard updates

Output path: `.ai/reports/[report-name]-YYYY-MM-DD-HH-MM.md`

Reports are your project's heartbeat.
