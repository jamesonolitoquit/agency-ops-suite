# .ai/ — AI Execution Framework

Kernel-like system enabling deterministic, high-quality autonomous code execution.

## System Architecture

```
.ai/
├── orchestrators/      # Task routing, context loading, execution coordination
├── roles/              # 8 specialized engineering personas
├── commands/           # 6 executable operational procedures
├── workflows/          # 5 multi-stage pipelines
├── standards/          # 7 non-negotiable engineering rules
├── scanners/           # 8 automated quality auditors
├── checklists/         # 4 operational validation checklists
├── context/            # 4 persistent project intelligence files
├── memory/             # 5 long-term engineering memory files
└── debug/              # 2 systematic debugging procedures
```

## Design Principles

- **Load by domain, not everything** — Context-router classifies task, loads only relevant context
- **Aggressive pruning** — Max 100KB context per session
- **Deterministic execution** — Same input → same output
- **No hallucinations** — All decisions grounded in standards/memory
- **Minimal boilerplate** — All files 300-500 words max
- **Operational logic only** — No motivational language

## Folder Guide

| Folder | Purpose | When Loaded |
|--------|---------|-------------|
| **orchestrators/** | Task routing engine, context limits, role coordination | Every session (auto) |
| **roles/** | 8 personas: architect, backend, security, performance, debug, etc | Task-dependent |
| **commands/** | Reusable procedures: plan-feature, review-pr, analyze-bug, etc | When task requires |
| **workflows/** | Multi-stage pipelines: build-feature, emergency-hotfix, etc | When executing workflow |
| **standards/** | Architecture, naming, API, security, component, UI, backend rules | Continuous (linting) |
| **scanners/** | 8 audit systems for quality, security, performance, etc | Triggered/scheduled |
| **checklists/** | Pre-flight validation for production, security, accessibility, launch | Before critical ops |
| **context/** | Project overview, stack, business rules, deployment | Session start |
| **memory/** | Decisions, failed approaches, optimizations, incidents, bugs | When pattern matches |
| **debug/** | Root-cause analysis, incident response procedures | When debugging/incident |

## Quick Start

1. **New task:** Describe what you want → context-router classifies → loads relevant files
2. **Feature planning:** Activate system-architect + backend-api-architect → run plan-feature command
3. **Bug investigation:** Activate senior-debug-engineer → follow root-cause-analysis procedure
4. **Production deployment:** Run production-checklist → security-checklist → launch-checklist
5. **Code review:** Activate ui-systems-engineer/backend-api-architect → run review-pr command

## Design Principles

### Maintainability
- Modular architecture with clear separation of concerns
- Predictable, consistent structure
- Self-documenting operations

### Scalability
- Extensible workflows and reusable commands
- Composable systems without overlap
- Role-based specialization

### Reliability
- Defensive engineering and validation-first logic
- Production-ready by default
- Explicit error handling

### Security
- Least privilege principles
- Secure defaults
- Proactive risk detection

### Performance
- Efficient execution paths
- Minimal operational complexity
- Optimized decision trees

### Clarity
- Explicit instructions with low ambiguity
- Deterministic outputs
- Human-readable procedures

## Important Constraints

**Do NOT:**
- Generate unnecessary complexity
- Introduce overlapping systems
- Create vague placeholder markdown
- Weaken architectural boundaries

**Every file added to `.ai/` must:**
- Have a clear, measurable purpose
- Integrate into the broader system
- Improve autonomous development capabilities
- Provide operational value

## Integration Points

### With VS Code Copilot
- Load context from `context/project-context.md` at session start
- Reference `standards/*` during code generation
- Execute `commands/` as Copilot Agent tasks

### With CI/CD
- Run scanners on every commit/PR
- Generate reports automatically
- Fail builds on critical issues (security, architecture)

### With Git Workflow
- Pre-commit hooks run workspace-hygiene scanner
- PR workflows trigger architecture and security scanners
- Deployment pipelines use checklists and readiness scanner
